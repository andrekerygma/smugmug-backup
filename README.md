# SmugMug Backup

Local app with a visual interface for backing up a SmugMug library to your computer, preserving gallery folder structure and showing live progress.

## What It Does

- Discovers galleries from the authenticated account through the official SmugMug API.
- Mirrors each gallery `UrlPath` into local folders.
- Downloads photos and videos while preserving the gallery structure.
- Runs multiple downloads in parallel to speed up the backup.
- Skips files that already exist when the size matches.
- Saves a local manifest so completed galleries are not processed again.
- Shows status, active progress, per-gallery summary, and a live log in the browser.

## Implementation Choice

Instead of automating temporary ZIP links generated by the site, this app downloads original files directly through the SmugMug OAuth API. In practice this makes the backup more stable, improves resume behavior, and avoids depending on temporary archive packages generated on demand.

## Prerequisites

- Node.js 24+
- SmugMug OAuth credentials:
  - `API key`
  - `API secret`

## How To Connect

1. Request an app at `https://api.smugmug.com/api/developer/apply`.
2. After approval, collect the `API key` and `API secret`.
3. Run the app locally.
4. Enter the `API key`, `API secret`, and destination folder.
5. Adjust `Parallel downloads` if you want more speed.
6. Click `Connect to SmugMug`.
7. Authorize the app in the browser.
8. The app automatically stores the `user token` and `user secret`.

## How To Run

```bash
npm install
npm run dev
```

Or:

```bash
npm start
```

If you downloaded the project as a ZIP from GitHub, `node_modules/` is not included. Run `npm install` once inside the project folder before using `npm run dev` or `npm start`.

Then open:

```text
http://localhost:4317
```

## Usage Flow

1. Enter the credentials and destination folder.
2. Adjust `Parallel downloads` if you want faster transfers.
3. Click `Save configuration`.
4. Click `Connect to SmugMug`.
5. Click `Scan galleries`.
6. Click `Start backup`.
7. If needed, use `Stop` to interrupt safely after the current file finishes.

## Local Persistence

- Configuration: `data/config.json`
- Last scan catalog: `data/catalog.json`
- Incremental manifest: `data/manifest.json`

Credentials are stored locally in plain text inside these data files.

Because of that, the `data/` folder should stay out of commits and out of any ZIP you share.

## Safe Sharing

- The repository already ignores generated files in `data/`.
- Before sharing the full folder, confirm that `data/` does not contain `config.json`, `catalog.json`, `manifest.json`, or any files inside `data/albums/`.
- If you have used the app locally, delete the contents of `data/` so you do not share OAuth credentials, account names, gallery names, archived URLs, or library file names.
